# GPS Map Camera Integration\n\nThis feature provides intelligent camera selection that automatically chooses between GPS Map Camera app and the normal device camera based on client configuration and app availability.\n\n## Features\n\n### Smart Camera Selection\n- **Automatic Detection**: Checks if GPS Map Camera app is installed\n- **Client Configuration**: Respects client GPS settings from server\n- **Fallback Support**: Uses normal camera when GPS camera is unavailable\n- **User Choice**: Allows manual selection when needed\n\n### Image Processing\n- **Map Overlay Removal**: Removes only map overlays while preserving measurement guides\n- **Board Size Preservation**: Keeps measurement overlays and board size indicators intact\n- **GPS Data Extraction**: Extracts location metadata from GPS camera photos\n\n### Integration Points\n- **MeasurementCamera**: Enhanced with GPS camera selection\n- **SmartCameraButton**: New component for easy integration\n- **RecceFormScreen**: Updated to use smart camera selection\n\n## Usage\n\n### Using SmartCameraButton Component\n\n```tsx\nimport SmartCameraButton from '../components/SmartCameraButton';\n\n<SmartCameraButton\n  clientId={clientId} // Optional: for GPS configuration check\n  photoType=\"front\" // 'front' | 'side' | 'closeUp'\n  width=\"24\" // Optional: for measurement overlay\n  height=\"18\" // Optional: for measurement overlay\n  onPhotoCapture={(photoUri, metadata) => {\n    // Handle captured photo\n    console.log('Photo captured:', photoUri);\n    console.log('Has GPS data:', metadata?.hasGPSData);\n  }}\n  title=\"Take Photo\" // Optional: custom button title\n/>\n```\n\n### Using Camera Selection Hook\n\n```tsx\nimport { useCameraSelection } from '../hooks/useCameraSelection';\n\nconst {\n  isCapturing,\n  selectCamera,\n  isGPSCameraAvailable,\n  checkGPSCameraAvailability\n} = useCameraSelection({\n  clientId: 'client123',\n  photoType: 'front',\n  onCapture: (photoUri, metadata) => {\n    // Handle photo capture\n  },\n  onError: (error) => {\n    // Handle errors\n  }\n});\n\n// Check GPS camera availability\nuseEffect(() => {\n  checkGPSCameraAvailability();\n}, []);\n\n// Trigger camera selection\n<TouchableOpacity onPress={selectCamera}>\n  <Text>Take Photo</Text>\n</TouchableOpacity>\n```\n\n### Direct Service Usage\n\n```tsx\nimport { cameraSelectionService } from '../services/cameraSelectionService';\n\n// Check if GPS Map Camera is installed\nconst isInstalled = await cameraSelectionService.isGPSMapCameraInstalled();\n\n// Select camera based on client configuration\nawait cameraSelectionService.selectCamera({\n  clientId: 'client123',\n  photoType: 'front',\n  onCapture: (photoUri, metadata) => {\n    // Handle captured photo\n  },\n  onError: (error) => {\n    // Handle error\n  }\n});\n```\n\n## Configuration\n\n### Client GPS Settings\n\nThe system checks client configuration to determine if GPS location overlay should be enabled:\n\n```typescript\n// Client configuration example\n{\n  enableLocationOverlay: true, // Enable GPS features for this client\n  mapSize: 60, // Map overlay size\n  showAddress: true, // Show address in overlay\n  showCoordinates: true, // Show coordinates\n  showTimestamp: true, // Show timestamp\n  position: 'bottom-left' // Overlay position\n}\n```\n\n### GPS Map Camera App\n\n**Android Package**: `com.gpsmapcamera.app`\n**iOS URL Scheme**: `gpsmapcamera://`\n\n## Behavior Logic\n\n### When Client Has GPS Enabled\n1. Check if GPS Map Camera is installed\n2. If installed: Open GPS Map Camera directly\n3. If not installed: Show install prompt with fallback to normal camera\n\n### When Client Has GPS Disabled\n1. Use normal device camera\n2. No GPS-related prompts or features\n\n### When No Client ID Provided\n1. Show manual selection dialog\n2. User can choose between GPS Map Camera or normal camera\n\n## Image Processing\n\n### Map Overlay Removal\n\nThe system automatically processes GPS Map Camera photos to:\n- Detect map overlay regions\n- Remove only the map overlay\n- Preserve measurement guides and board size indicators\n- Keep text overlays and annotations\n\n### Metadata Handling\n\n```typescript\n// Photo metadata structure\n{\n  hasGPSData: boolean,\n  source: 'gps_map_camera' | 'normal_camera',\n  capturedAt: string,\n  locationData?: {\n    latitude: number,\n    longitude: number,\n    address: string\n  },\n  measurements?: {\n    width: number,\n    height: number,\n    unit: string\n  }\n}\n```\n\n## Error Handling\n\n### Common Scenarios\n- **GPS Camera Not Installed**: Shows install prompt with fallback\n- **GPS Camera Fails to Open**: Automatically falls back to normal camera\n- **Permission Denied**: Shows permission request with settings link\n- **Image Processing Fails**: Returns original image\n\n### Error Messages\n- Clear, user-friendly error messages\n- Automatic fallback to normal camera when possible\n- No disruption to user workflow\n\n## Installation Requirements\n\n### Dependencies\n- `react-native-image-picker`: For normal camera functionality\n- `react-native-fs`: For image processing and storage\n- `@react-native-community/geolocation`: For location services\n\n### Permissions\n- Camera permission (both apps)\n- Location permission (for GPS features)\n- Storage permission (for image processing)\n\n## Testing\n\n### Test Scenarios\n1.
