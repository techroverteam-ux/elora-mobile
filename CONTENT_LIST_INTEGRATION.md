# Content List Integration Guide\n\nThis guide explains how to use the new enhanced content list components that provide the same UI and features as your category data list.\n\n## Components Created\n\n### 1. `EnhancedContentList.tsx`\nThe main component that displays content list with enhanced UI, video player integration, and media type handling.\n\n### 2. `ContentList.tsx` \nA simpler version of the content list component without video player integration.\n\n### 3. `useContentList.ts`\nA custom hook that handles content list interactions and media type handling.\n\n### 4. `contentListUtils.ts`\nUtility functions for content list handling, validation, and transformation.\n\n## API Changes Made\n\n### Backend (API)\n1. **Added `contentList` field to category query**: The mobile categories API now returns the `contentList` field from the database.\n2. **Enhanced video streaming**: Improved the Azure blob controller to handle video streaming with proper range requests and CORS headers.\n\n### Frontend (Mobile App)\n1. **Updated `ContentListView.tsx`**: Now uses the enhanced content list component.\n2. **Enhanced `CategorieDataList.tsx`**: Better content list detection and navigation logic.\n\n## Usage Examples\n\n### Basic Usage\n```tsx\nimport EnhancedContentList from '../components/EnhancedContentList';\n\nconst MyScreen = () => {\n  const contentList = [\n    {\n      title: \"Video Tutorial\",\n      subtitle: \"Learn the basics\",\n      order: 1,\n      mediaUrl: \"https://example.com/video.mp4\",\n      mediaType: \"video\"\n    },\n    {\n      title: \"Audio Lesson\",\n      subtitle: \"Listen and learn\",\n      order: 2,\n      mediaUrl: \"https://example.com/audio.mp3\",\n      mediaType: \"audio\"\n    }\n  ];\n\n  return (\n    <EnhancedContentList\n      contentList={contentList}\n      headerTitle=\"My Content\"\n    />\n  );\n};\n```\n\n### With Custom Handlers\n```tsx\nimport { useContentList } from '../hooks/useContentList';\n\nconst MyScreen = () => {\n  const { handleItemPress } = useContentList({\n    onVideoPress: (item) => {\n      // Custom video handling\n      console.log('Playing video:', item.title);\n    },\n    onAudioPress: (item) => {\n      // Custom audio handling\n      console.log('Playing audio:', item.title);\n    }\n  });\n\n  // ... rest of component\n};\n```\n\n### Using Utility Functions\n```tsx\nimport { \n  transformApiContentList, \n  getContentStats, \n  sortContentByOrder \n} from '../utils/contentListUtils';\n\n// Transform API response\nconst contentList = transformApiContentList(apiResponse.contentList);\n\n// Get statistics\nconst stats = getContentStats(contentList);\nconsole.log(`Total: ${stats.total}, Videos: ${stats.video}, Audio: ${stats.audio}`);\n\n// Sort by order\nconst sortedList = sortContentByOrder(contentList);\n```\n\n## Content List Data Structure\n\n```typescript\ninterface ContentListItem {\n  title: string;           // Required: Display title\n  subtitle?: string;       // Optional: Subtitle text\n  url?: string;           // Optional: External URL to open\n  order: number;          // Required: Display order\n  mediaUrl?: string;      // Optional: Media file URL\n  mediaType?: 'image' | 'video' | 'audio' | 'file'; // Optional: Media type\n}\n```\n\n## Features\n\n### 1. **Enhanced UI**\n- Same styling as category list\n- Media type icons and colors\n- Order badges\n- Statistics bar\n- Gradient backgrounds\n\n### 2. **Video Player Integration**\n- Full-screen video modal\n- Uses your existing `EnhancedVideoPlayer` component\n- Proper video streaming support\n\n### 3. **Media Type Handling**\n- **Video**: Opens in full-screen video player\n- **Audio**: Can be integrated with your audio player\n- **Image**: Can be integrated with image viewer\n- **File**: Can be integrated with file viewer\n- **URL**: Opens external links\n\n### 4. **Interactive Features**\n- Pull-to-refresh support\n- Loading states\n- Error handling\n- Empty state display\n\n## Integration Steps\n\n### 1. **API Integration**\nThe API changes have been made to return `contentList` in the categories response. Make sure to deploy the updated API.\n\n### 2. **Component Usage**\nReplace your existing content list implementations with the new `EnhancedContentList` component.\n\n### 3. **Navigation Updates**\nUpdate your navigation logic to check for `contentList` availability and navigate to the enhanced content list view.\n\n### 4. **Customization**\nCustomize the media type handlers based on your app's requirements:\n\n```tsx\nconst customHandlers = {\n  onVideoPress: (item) => {\n    // Integrate with your video player\n    navigation.navigate('VideoPlayer', { videoUrl: item.mediaUrl });\n  },\n  onAudioPress: (item) => {\n    // Integrate with your audio player\n    AudioPlayer.play(item.mediaUrl);\n  },\n  onImagePress: (item) => {\n    // Integrate with your image viewer\n    navigation.navigate('ImageViewer', { imageUrl: item.mediaUrl });\n  }\n};\n```\n\n## Troubleshooting\n\n### Content List Not Showing\n1. Check if the API is returning `contentList` in the response\n2. Verify the data structure matches the expected format\n3. Check console logs for debugging information\n\n### Video Not Playing\n1. Ensure the video URL is accessible\n2. Check Azure blob storage configuration\n3. Verify CORS settings for video streaming\n\n### Styling Issues\n1. Check if all required dependencies are installed\n2. Verify Expo vector icons are properly configured\n3. Ensure LinearGradient is available\n\n## Dependencies Required\n\n```json\n{\n  \"@expo/vector-icons\": \"^13.0.0\",\n  \"expo-linear-gradient\": \"^12.0.0\",\n  \"react-native-video\": \"^5.2.1\"\n}\n```\n\n## Next Steps\n\n1. **Test the integration** with your existing content\n2. **Customize the styling** to match your app's theme\n3. **Add more media type handlers** as needed\n4. **Implement caching** for better performance\n5. **Add offline support** for downloaded content\n\nThe enhanced content list now provides the same rich UI and functionality as your category data list, with proper video player integration and media type handling.
